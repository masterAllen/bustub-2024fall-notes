# Project4.1 前言


没想到，七个多月之后，有心情有时间来把之前没做完的 CMU 15455 收尾了。不同的是，以前的三个任务，我是参考别人博客加上和 ChatGPT 对话，手动编码，然后不断地去 DEBUG 来完成实现的；简单来说：古法编程，不对，应该叫古法向现代的过渡编程，毕竟也用了很多 AI。这一次的任务四，我直接尝试 vibe coding，正好 GPT 5.4 出来有一会了，这次就用 GPT 5.4 来进行 vibe coding。

结果让我的后背发凉，一种深深的恐惧感。任务完成的相当顺利，我百分百可以说：任务的完成没有我的任何一句贡献代码（如果注释不算的话）。全是 AI 完成，外加项目本身极其卓越的测试系统。实在太过可怕了，我只是将官网的实验指导，逐步地去复制粘贴给 AI，它很顺利地一步步完成。虽然并不是一次就完美通过，但是把测试结果告诉它后，两三次就可以定位并解决问题。甚至最后线上评测中，我把出错的内容喂给它，其中并没有什么特别关键的输出信息，它居然也能凭借着只言片语猜测成功并解决。

依照 Project2 的经验（网上许多说 P2 和 P4 难度相当），我设定的完成目标是 50h，这已经算是比较严格了。没想到，最终我是前一天的晚上十点开始，做到当天夜里一点，第二天从十点开始，做到当天的五点。没错，只用了 10 小时！！！而且，大部分时间是我去尝试理解，以防后面失控了看不懂代码，如果真的纯托管，两个小时或许就可以了！！

太让人恐惧的结果了。人在机器面前真的太过渺小，以前虽然也有很多是托管给 AI，但我感觉都是重复枯燥的、简单基础的；这一次对于这一个计算机学生相当有口碑的难项目，直接秒杀。这可是有并发的测试啊！

我的 Github 名字叫做 masterAllen，这里的 master 就因为 AI，当时 2016 年末，野狐围棋上突然出现一个叫做 master 的用户，印象中已经 60 连胜，之后又不断战胜柯洁等人，最后注销了。这给了我相当大的印象，所以不久之后注册 Github 时，选择了 master。如今，曾经的围棋手现在成了我自己，我体会到当时从业者的绝望和无奈。人定胜天，多么可笑。未来，究竟路在何方。
